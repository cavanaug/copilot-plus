<objective>
Extend the `copilot-cli` wrapper with per-project config support and `COPILOT_ARGS` export using TDD.

Purpose: Allow projects to place a `.copilot/config.json` in their root directory to inject project-specific CLI flags. The project config merges with the global config additively, with key-level override (project wins when same key appears in both). After assembling the full set of config-injected flags, export them as `COPILOT_ARGS` (shell-quoted, space-separated) so nested copilot invocations can use `copilot $COPILOT_ARGS` to inherit the same launch context.

<!-- Merge strategy: additive with key-level override -->
Given:
  global:  { "--allow-tool": ["shell(sed)"], "--model": "gpt-4.1" }
  project: { "--allow-tool": ["shell(git:*)"], "--add-dir": ["/tmp"] }

Result:
  --model gpt-4.1          ← global (not overridden by project)
  --allow-tool shell(git:*) ← project overrides global for this key
  --add-dir /tmp            ← project only (additive)

Flag order: global flags first (minus overridden keys), then project flags.

<!-- Key-level override algorithm -->
1. Parse project config, collect all -- keys into a set: project_keys
2. Process global config: for each -- key, SKIP if key is in project_keys
3. Process project config: inject all -- keys normally
4. Result: global (non-overridden) flags + project flags

  <files>copilot-cli</files>
  <action>
    Refactor `copilot-cli` to support two config sources with key-level override merge.

    **New implementation:**

    ```bash
    #!/usr/bin/env bash
    set -euo pipefail

    GLOBAL_CONFIG="${HOME}/.copilot/config.json"
    PROJECT_CONFIG="${PWD}/.copilot/config.json"

    # Read all -- flags from a config file into a named array (by reference)
    # Usage: read_config_flags <config_file> <array_ref> [<skip_keys_array_ref>]
    # skip_keys_array_ref: associative array of keys to skip (for key-level override)
    read_config_flags() {
      local config_file="$1"
      local -n _flags_ref="$2"
      local -n _skip_ref="$3"

      [[ -f "$config_file" ]] && [[ -r "$config_file" ]] || return 0

      if ! jq empty "$config_file" 2>/dev/null; then
        echo "copilot-wrapper: error: invalid JSON in $config_file" >&2
        exit 1
      fi

      while IFS= read -r key; do
        # Key-level override: skip keys that appear in project config
        [[ -v _skip_ref["$key"] ]] && continue

        local vtype
        vtype=$(jq -r --arg k "$key" '.[$k] | type' "$config_file")

        case "$vtype" in
          array)
            while IFS= read -r element; do
              _flags_ref+=("$key" "$element")
            done < <(jq -r --arg k "$key" '.[$k][]' "$config_file")
            ;;
          boolean)
            local val
            val=$(jq -r --arg k "$key" '.[$k]' "$config_file")
            [[ "$val" == "true" ]] && _flags_ref+=("$key")
            ;;
          string|number)
            local val
            val=$(jq -r --arg k "$key" '.[$k]' "$config_file")
            _flags_ref+=("$key" "$val")
            ;;
          object|null|*)
            echo "copilot-wrapper: error: unsupported value type '$vtype' for key '$key' in $config_file" >&2
            exit 1
            ;;
        esac
      done < <(jq -r 'keys[] | select(startswith("--"))' "$config_file")
    }

    # Collect project keys for key-level override (skip map for global processing)
    declare -A project_keys=()
    if [[ -f "$PROJECT_CONFIG" ]] && [[ -r "$PROJECT_CONFIG" ]]; then
      if ! jq empty "$PROJECT_CONFIG" 2>/dev/null; then
        echo "copilot-wrapper: error: invalid JSON in $PROJECT_CONFIG" >&2
        exit 1
      fi
      while IFS= read -r key; do
        project_keys["$key"]=1
      done < <(jq -r 'keys[] | select(startswith("--"))' "$PROJECT_CONFIG")
    fi

    # Build combined flags: global (minus overridden keys) + project
    config_flags=()
    declare -A no_skip=()
    read_config_flags "$GLOBAL_CONFIG" config_flags project_keys
    read_config_flags "$PROJECT_CONFIG" config_flags no_skip

    # Export COPILOT_ARGS: shell-quoted, space-separated config-injected flags only (not user args)
    # This allows nested calls: copilot $COPILOT_ARGS <new-args>
    COPILOT_ARGS=$(printf '%q ' "${config_flags[@]+"${config_flags[@]}"}") || true
    COPILOT_ARGS="${COPILOT_ARGS% }"  # trim trailing space
    export COPILOT_ARGS

    exec copilot "${config_flags[@]+"${config_flags[@]}"}" "$@"
    ```

    Notes:
    - `local -n` is bash 4.3+ nameref — available on all modern Linux systems
    - `[[ -v _skip_ref["$key"] ]]` checks associative array membership without triggering set -u
    - Project config is validated twice (once for key collection, once in read_config_flags) — acceptable since it's only done at invocation time; alternatively, restructure to validate once. Use whichever is cleaner.
    - `declare -A no_skip=()` is an empty skip map passed when processing project config (nothing to skip)
    - `printf '%q '` shell-quotes each element of config_flags for safe use in `$COPILOT_ARGS`
    - `export COPILOT_ARGS` makes it available to the exec'd copilot process and any subprocesses it spawns
